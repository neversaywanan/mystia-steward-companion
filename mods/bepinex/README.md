# BepInEx Mod 使用说明

本文档仅说明 Mod 安装、使用、配置和故障排查。开发环境、构建流程、运行时反射和本地 API 说明见 [README.dev.md](README.dev.md)。

## 安装 BepInEx

1. 找到游戏根目录。Steam 可通过“管理 -> 浏览本地文件”打开，目录中应能看到游戏 `.exe` 和 `GameAssembly.dll`。
2. 打开 BepInEx Bleeding Edge 下载页：<https://builds.bepinex.dev/projects/bepinex_be>。
3. 下载与系统和游戏架构匹配的 Unity IL2CPP 包。Windows 64 位通常选择：

```text
BepInEx-Unity.IL2CPP-win-x64-6.0.0-be.*.zip
```

4. 将压缩包内容解压到游戏根目录。解压后应出现：

```text
BepInEx/
doorstop_config.ini
winhttp.dll
```

5. 启动游戏一次，等待 BepInEx 生成目录和 IL2CPP interop 程序集。首次启动可能较慢。
6. 关闭游戏，确认以下目录存在：

```text
BepInEx/config/
BepInEx/core/
BepInEx/interop/
BepInEx/plugins/
```

如果没有生成这些目录，通常是下载了非 IL2CPP 包、解压位置错误，或游戏没有成功启动过一次。

## 安装 Mod

获取安装包：

```text
mystia-steward-companion-bepinex.zip
```

将压缩包中的 `mystia-steward-companion/` 整个目录放入游戏目录：

```text
游戏根目录/
  BepInEx/
    plugins/
      mystia-steward-companion/
        MystiaStewardCompanion.BepInEx.dll
        companion/
          mystia-steward-companion.exe
```

安装后启动游戏。控制台或 `BepInEx/LogOutput.log` 中应出现 `mystia-steward-companion loaded` 和本地 API 启动日志。

## 使用方式

- `F8`：在游戏与独立伴随窗口之间切换焦点；窗口隐藏或关闭到托盘时会重新显示。
- `F10`：开启或关闭伴随窗口鼠标穿透锁定；穿透时鼠标点击会落到下方游戏或其他窗口。
- `RS Click`：手柄默认在游戏与独立伴随窗口之间切换焦点。
- 伴随窗口获得焦点后可用手柄操作：左摇杆/十字键移动焦点，`A` 确认，`B` 返回，`LB/RB` 切换页面，`LT/RT` 滚动。
- 在推荐行上按 `A` 或 `X` 可收藏/取消收藏当前料理或酒水；`Y` 可进入稀客订单专注模式，专注模式内按 `Y` 切换精简模式。
- `概览`：通过 `状态`、`库存`、`操作` 页签分别显示连接状态、运行时数据、低库存和快捷键；开启 `设置 -> 显示调试信息` 后会额外显示 API、运行时来源和快照耗时。
- `普客`：按当前地区显示料理和酒水推荐。
- `稀客`：按当前经营场景、候选稀客和点单词条显示满足点单的推荐，推荐顺序可在 `设置` 页调整。
- `经营中`：顶部显示经营场景、推荐数据、已摆放厨具和目标厨具；下方通过 `稀客` / `普客` 页签分区。`稀客` 页查看当前稀客、携带金钱、桌位、点单词条、满足点单的推荐料理、推荐加料和推荐酒水；订单默认按点单先后排列，也可在 `设置` 页切换为稀客分组，严格满足点单的推荐不足时，会额外显示明确标注的“喜好备选（不满足点单）”。如果运行时缓存偶发留下过时稀客订单，可在订单行右侧点击删除按钮清理该笔缓存。`普客` 页显示普客订单；开启实验性自动化总开关和“启用普客处理”后，可按普客独立阶段开关自动处理最早出现的未满足普客订单。开启 `设置 -> 显示调试信息` 后会额外显示扫描状态、快照耗时、订单来源和内部诊断字段。
- 料理和酒水推荐行可点击 `收藏`。同一稀客再次出现相同料理或酒水点单词条时，可用收藏会优先显示；再次点击可取消收藏。
- `稀客订单专注模式`：只显示当前点单推荐；可切换精简模式，并自行设置推荐料理/推荐酒水显示数量，没有点单时显示等待提示。
- `任务`：读取当前进度中可接取、进行中或可完成的交互任务，默认显示可接取和可完成任务，可用多选按钮切换筛选状态；任务行会显示标题、角色、任务状态，并尽量显示未接取 NPC 所在场景。开启 `设置 -> 显示调试信息` 后，可额外查看任务 ID、读取来源和扫描状态。稀客邀请区会读取日间场景 NPC，并可在当前场景和全部场景之间切换；随后检查游戏原生羁绊等级、今晚已邀请状态和成功邀请对话包。可邀请项支持单独邀请或一次邀请全部，已邀请、羁绊不足、当前不可见或缺少成功邀请对话的候选会保留显示但禁用按钮。它不会直接刷出稀客，后续出现与否仍由游戏自己的经营流程处理。
- 独立伴随窗口会保存上次的位置和大小。重新启动游戏和 Mod 后，会优先恢复到上次调整过的窗口状态。
- `设置`：通过 `窗口`、`推荐`、`排序`、`自动化` 页签管理窗口背景透明度（20% - 100%）、焦点切换行为、切换冷却时间、始终置顶、鼠标穿透锁定、主题、手柄导航、调试信息显示、缺失厨具过滤、任务料理优先、经营中订单排序、料理/酒水推荐排序、实验性游戏界面置顶、目标厨具高亮、实验性自动化总开关和自动化参数。`显示调试信息` 默认关闭；开启后会额外出现 `调试` 页签，并显示日志页、BepInEx 原生日志窗口控制、扫描状态、运行时来源、性能耗时和订单内部来源。
- `修改`：用 `-10`、`+10` 和 `99` 快捷按钮修改当前运行时材料和酒水库存数量，也可一键把当前存档可编辑材料或酒水设为 `99`；修改后需要在游戏内保存才会持久化。
- `帮助`：使用折叠面板展示快捷键、窗口、推荐、经营中、自动化、任务、修改和日志排查说明。帮助内容保存在 `apps/companion/src/data/help-content.json`，构建前可直接修改。
- `日志`：仅在 `设置 -> 显示调试信息` 开启后显示。可读取 `BepInEx/LogOutput.log` 尾部内容、按需开启经营诊断，并可打开日志文件夹；日志读取有行数和字节上限，不会在窗口内无限累积。

如果游戏还停留在标题、菜单或加载页面，伴随窗口会提示运行时数据不可用。进入游戏并加载进度后，Mod 会自动读取当前游戏状态，不需要选择存档文件。

进入夜间经营后，Mod 会尝试读取当前场景已摆放的全部厨具。`排除缺失厨具` 开启时，推荐列表会隐藏当前场景没有对应厨具、无法制作的料理；如果未读取到厨具快照，则不会进行过滤。

经营中订单排序默认使用 `点单顺序`；切换为 `稀客分组` 后，同一稀客的多笔订单会放在一起，稀客组之间仍按该稀客最早订单出现时间排列。该设置会影响 `经营中`、稀客订单专注模式、游戏界面置顶目标和自动化处理的当前第一笔订单。

料理排序默认按 `满足点单 Tag -> 料理分数 -> 加料数量 -> 资源压力 -> 料理售价 -> 加料成本 -> 料理 ID` 执行；酒水排序默认按 `满足点单 Tag -> 酒水分数 -> 酒水售价 -> 酒水 ID` 执行。可在 `设置` 页启用、关闭、调整升降序或移动排序项。推荐排序会影响 `稀客`、`经营中`、稀客订单专注模式，以及自动化处理的当前第一笔订单。

`优先任务料理` 默认关闭。开启后，若当前稀客存在“经营中请该角色品尝指定料理”的已接取任务，`经营中` 和稀客订单专注模式会把该任务料理优先放到当前订单推荐第一位，并在推荐行显示 `任务` 标识；该功能只调整推荐顺序，不会自动触发或完成任务。

`修改` 页会直接写入当前游戏运行时库存。建议在非夜间经营流程中使用；若在经营中修改，结果可能与实时消耗同时发生。

收藏数据保存在 `BepInEx/config/MystiaStewardCompanion/favorites.json`。更新 Mod 时通常只替换 `BepInEx/plugins/mystia-steward-companion/` 下的文件，不会覆盖该收藏文件。

## 自动化（实验性）

自动化默认关闭。需要使用时，先到 `设置` 页开启 `启用自动化（实验性）`，再到 `经营中` 页分别选择稀客订单和普客订单的具体子功能。两类订单的阶段开关互不影响。

`设置` 页还可以调整自动化参数：稀客并发默认 `2`，普客并发默认 `3`，稀客等待送餐盘默认 `30` 秒，普客保温箱复查默认 `45` 秒，最大重试默认 `3` 次，最大回退默认 `2` 次。参数会在下一轮自动化轮询生效；并发过高可能抢占厨具，等待时间过短可能导致重复开锅。

稀客订单可配置：

- `自动完成订单`：当送餐盘中存在满足稀客订单的料理和酒水时，尝试完成该订单；为降低状态错乱风险，每轮最多完成一笔。
- `自动取酒`：按稀客订单的推荐酒水，从运行时库存放入送餐盘。
- `自动开始料理`：按稀客订单的推荐料理，选择可用厨具开始制作。
- `自动收取料理`：料理完成后尝试收入送餐盘。
- `只处理收藏配方`：只使用已收藏的料理方案和酒水。
- `出错时暂停`：遇到非临时错误后暂停当前订单，直到订单变化或重新开启。

普客订单需要先开启 `启用普客处理`，再单独配置：

- `自动开始料理`：按普客订单所需料理选择可用厨具开始制作。
- `自动收取料理`：料理完成后收至游戏料理暂存容器，不占用玩家送餐盘，也不直接送达订单。
- `出错时暂停`：遇到非临时错误后只暂停对应普客订单，不影响稀客自动化或其他普客订单。

开启任一 `自动开始料理` 后，Mod 会固定尝试完成原生 QTE 奖励结算。该流程不会打开游戏音游面板；若游戏版本变动导致入口不可用，会在自动化状态中显示诊断信息，但不会中断已开始的料理。

自动化按伴随窗口当前排序处理稀客候选；为了避免一次轮询对多个运行时订单同时写入，`自动完成订单` 每轮最多完成 1 笔。稀客料理和酒水会分别送达：送餐盘中先出现料理或酒水时，Mod 会先把该项写入订单并从送餐盘移除，等另一项补齐后再触发完整评价。稀客订单没有满足点单 Tag 的料理时，自动化会改用“喜好备选”料理，并在状态中标记。普客会按首次出现顺序并发处理未满足订单，每笔订单独立记录开始料理、收至保温箱和暂停状态；识别到新普客订单或订单状态变化后会立即触发一轮处理，重复轮询仍有节流保护。稀客和普客会共用本轮厨具预约表：Mod 会按当前已摆放厨具数量限制同类厨具的开锅请求，并在同类厨具容量不足时优先给待处理普客订单保留位置，稀客料理进入等待态但不会计作失败。经营中页会显示厨具预约和送餐盘压力，便于判断自动化是否因为厨具或送餐盘占用而等待。已经进入制作中的普客料理会绑定到目标订单和桌位，后续轮询不会因为同类厨具存在多个、同桌换单或同菜多单而重复开锅。普客自动化不处理酒水，也不会把料理直接送达订单；料理收至保温箱后，需要玩家按游戏原生流程送达。厨具占用、目标成品暂未进入容器、料理暂存容器不可用、运行时对象暂不可读等临时状态会继续等待并重试；稀客目标料理或酒水长时间未出现时会回退到上一步重新执行，超过回退上限后才按设置暂停。普客料理长时间未进入暂存容器时会重新确认后端待收取任务；若后端已完成收取，会用短期完成回执更新伴随窗口状态，必要时才重新开锅，不再因为等待暂存超时单独暂停。材料不足、无可用推荐、收藏限定不满足等情况视为硬错误；建议先在可回退的存档中测试。

稀客订单进入自动化后，会锁定本次选择的料理、加料和酒水，避免库存变化导致后续轮询切换到另一道推荐。若送餐盘中已有同一订单可接受的料理且持续堆积超过约 30 秒，Mod 会优先复用该料理，减少送餐盘中历史料理长期占位。

稀客自动化状态会按订单显示当前步骤、料理/酒水处理阶段、重试次数、回退次数和最近原因。若单笔订单暂停，可点击 `重试` 继续下一轮处理；若状态明显过期，可点击 `重置` 让该订单重新判断料理、酒水和完成条件。

若游戏 Debuff 导致厨具冻结，自动化会把已开锅料理视为待收取任务继续等待，不会重复开同一道目标料理；完成订单前会同时校验送餐盘中的料理和酒水，料理回退时也会重新校验酒水状态。

`游戏界面置顶推荐（实验性）` 开启后，伴随窗口会把当前第一笔稀客订单的第一条推荐料理、所需材料和推荐酒水发送给 Mod。打开游戏的料理选择面板或酒水架时，Mod 会尝试把这些条目排到列表前方；该功能只调整列表顺序，不自动点击，也不修改库存。若游戏版本导致字段名变化，置顶可能失效，但不会影响推荐或自动化流程。

`目标厨具高亮（实验性）` 可独立开启。开启后，当前第一笔稀客订单的推荐料理如果有目标厨具，Mod 会尝试让经营场景中对应已摆放厨具显示黄色脉冲高亮，便于快速定位；该功能只改变视觉提示，不会开始制作或修改库存。

## 独立伴随窗口

伴随窗口会在 Mod 加载后自动启动。它是一个独立桌面窗口，可以移动、缩放和置顶，不受游戏窗口边界限制。

如果在 `设置 -> 窗口` 开启 `鼠标穿透锁定`，伴随窗口会忽略鼠标点击，便于把窗口悬浮在游戏上方查看但继续操作游戏。需要恢复窗口操作时，按 `F10` 关闭穿透；也可以按 `F8` / `RS Click` 重新唤起窗口，或从托盘菜单选择 `关闭鼠标穿透`。

伴随窗口需要从游戏启动参数接收本地 API Token。若直接双击 `mystia-steward-companion.exe`，窗口会停留在未授权状态，不会持续高频重连；需要连接游戏时，从游戏内按 `F8` 或托盘菜单唤起。修改 `IP:Port` 后需要点击 `连接` 才会应用，连接失败时会逐步放慢重试，也可以点击 `停止` 暂停自动重连。

窗口关闭按钮默认隐藏到系统托盘，而不是直接退出。可以通过以下方式重新打开：

- 在游戏内按 `F8` 或 `RS Click`。
- 使用系统托盘菜单 `显示 mystia-steward-companion`。
- 再次双击 `mystia-steward-companion.exe`。

如果独立窗口当前获得焦点，再按 `F8` 或 `RS Click` 会按 `设置 -> 焦点切换` 的配置切回游戏：默认会隐藏伴随窗口，也可以改为保持伴随窗口悬浮、只切回游戏焦点。保持悬浮模式更适合配合透明度常驻查看；若游戏使用独占全屏，置顶效果可能受 Windows 窗口管理限制，建议使用窗口化或无边框窗口化。

`F8`、`RS Click`、托盘 `显示` 和托盘 `重连游戏` 都会自动关闭鼠标穿透锁定，避免窗口被找回后仍无法用鼠标操作。

游戏关闭或 Mod 卸载时，插件会立即通知伴随窗口退出；本地 API 断开检测仍作为兜底，避免游戏结束后残留后台窗口。

## 常用配置

配置文件位于：

```text
BepInEx/config/com.tyukki.mystia-steward-companion.cfg
```

如果从旧版本升级，Mod 会在新配置不存在时自动复制旧配置到上述新文件名。

常用项：

- `ToggleKey`：游戏与独立窗口焦点切换热键，默认 `F8`。
- `ControllerToggleKey`：手柄焦点切换热键，默认 `JoystickButton9`，常见映射为 `RS Click`。
- `Companion.AutoLaunch`：是否自动启动独立伴随窗口，默认开启。
- `Companion.ExecutablePath`：伴随窗口可执行文件路径；留空时自动从 Mod 目录和 `companion/` 子目录查找。
- `LocalApi.Port`：本地 API 端口，默认 `32145`。
- `LocalApi.ExposeLogs`：是否允许伴随窗口读取 `BepInEx/LogOutput.log`，默认开启；伴随窗口开启 `设置 -> 显示调试信息` 后可在 `日志` 页切换。
- `LocalApi.MaxLogLines`：日志页每次最多返回的日志行数，默认 `300`。
- `LocalApi.MaxLogBytes`：日志页每次最多扫描的日志尾部字节数，默认 `262144`。
- `BepInEx.DisableConsoleLogWindow`：启动后写入 `BepInEx/config/BepInEx.cfg`，将 BepInEx 控制台日志窗口设为下次启动关闭，默认开启；伴随窗口开启 `设置 -> 显示调试信息` 后可在 `设置 -> BepInEx` 切换。
- `BepInEx.HideConsoleWindow`：Mod 加载后尝试隐藏当前 Windows 控制台窗口，默认开启；设置页切换时也会尝试即时显示或隐藏当前窗口。
- `Diagnostics.EnableNightBusinessDiagnostics`：是否写入夜间经营诊断日志，默认关闭；伴随窗口开启 `设置 -> 显示调试信息` 后可在 `日志` 页切换。
- `SetConsoleUtf8`：加载 Mod 后尝试将 Windows 控制台切换到 UTF-8，默认开启。

开启经营诊断后，除 `night-business-diagnostics.log` 外，还会写入运行时固定数据快照：

- `runtime-static-data.log`：稀客事件变体映射表和运行时同名别名归一化来源。
- `runtime-tags.log`：料理/酒水标签文本和 TagRule。
- `runtime-database-diff.log`：食材、酒水、料理、菜品运行时数据与本地数据对照，并记录运行时表读取方式。
- `runtime-guests.log`：普客、稀客、映射稀客和特殊料理规则。
- `runtime-izakayas.log`：经营场景、场景标签和客人池。

这些文件不是游戏启动瞬间立即生成，而是在诊断开启且运行时经营数据扫描触发时读取；若游戏数据库尚未初始化，Mod 会每 5 秒重试。日志顶部 `Complete: True` 且 `Status` 中各项计数大于 0，表示本次固定数据读取成功。

自动化开锅、pending 收取和回退诊断会额外写入 `BepInEx/config/MystiaStewardCompanion/automation-jobs.log`，文件超过约 1 MB 时自动轮换为 `.1`。伴随窗口 `日志` 页会读取该文件并显示结构化自动化作业记录，也可以点击 `导出诊断包` 生成包含当前 snapshot 和日志尾部内容的 zip；排查“已开始制作但未进入送餐盘/保温箱”时优先提供诊断包或该页面内容。

## 故障排查

- `F8` 无法打开独立窗口：确认 `mystia-steward-companion.exe` 位于 `BepInEx/plugins/mystia-steward-companion/companion/`，或在 `Companion.ExecutablePath` 中填写绝对路径。若窗口已打开，`F8` 会在游戏和独立窗口之间切换。
- 伴随窗口鼠标点不到：通常是开启了鼠标穿透锁定。按 `F10` 关闭，或通过托盘菜单选择 `关闭鼠标穿透`。
- 一直显示运行时数据不可用：先确认已经进入游戏并加载进度；再到 `设置` 页开启 `显示调试信息`，进入 `日志` 页临时开启日志读取。
- `经营中` 没有稀客或稀客点单：先到 `设置` 页开启 `显示调试信息`，再进入 `日志` 页开启经营诊断并查看扫描状态，确认游戏内确实处于夜间经营流程。
- 启动时仍短暂出现控制台：Mod 加载时控制台已经由 BepInEx 创建，本次启动只能尝试隐藏；`BepInEx.DisableConsoleLogWindow=true` 写入的配置会在下一次启动生效。需要临时查看原生控制台时，可先开启 `设置 -> 显示调试信息`，再到 `设置 -> BepInEx` 开启。
- 控制台早期中文乱码：Mod 只能在自身加载后切换 UTF-8，不能修复 BepInEx preloader 已经输出的日志。日常建议关闭控制台；排查原生控制台问题时再临时开启。
排查运行时识别问题时，请先开启 `设置 -> 显示调试信息`，再在 `日志` 页开启日志读取和经营诊断，然后提供 `BepInEx/LogOutput.log`、诊断日志和伴随窗口 `日志` 页内容。
