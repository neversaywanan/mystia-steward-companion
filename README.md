# mystia-steward-companion

**mystia-steward-companion** 是面向《东方夜雀食堂》的 BepInEx IL2CPP Mod。它读取游戏当前运行时数据，并通过独立桌面伴随窗口提供普客、稀客和夜间经营推荐。

项目由游戏内 Mod 读取运行时数据，并通过本地回环 API 提供给桌面伴随窗口展示和操作。

## 功能概览

### 实时数据与推荐

- 实时读取已解锁料理、酒水、食材、库存、流行标签、明星店状态和当前经营场景。
- 普客页按当前地区推荐料理与酒水。
- 稀客页按候选稀客和点单 tag 推荐料理、加料与酒水，可在设置页调整推荐排序规则和同基础料理展示数量。
- 经营中页自动检测当前稀客点单，显示桌号、料理 tag、酒水 tag、当前稀客携带金钱和推荐结果。
- 经营中订单可按点单顺序或稀客分组排列；严格推荐不足时会额外显示明确标注的喜好备选。
- 支持收藏稀客常用料理方案和酒水；同一稀客再次出现相同点单 tag 时会优先显示可用收藏。

### 任务与稀客邀请

- 任务页可读取当前进度中可接取、进行中或可完成的交互任务，默认显示可接取和可完成任务。
- 任务状态支持多选筛选，并尽量显示未接取 NPC 所在场景；任务 ID 和读取来源默认隐藏，可用 `显示额外信息` 辅助排查。
- 任务页会列出日间场景中的稀客邀请候选，可在当前场景和全部场景之间切换，并可按羁绊等级筛选。
- 可邀请项支持单独邀请或一次邀请全部，不满足羁绊或已邀请的项会显示原因并禁用按钮；当前已邀请稀客会单独汇总显示。

### 经营辅助与自动化

- 支持稀客订单专注模式，可调节料理/酒水显示数量并切换精简模式，减少滚动切换。
- 设置页提供实验性自动化总开关和自动化参数；开启后可在经营中页分别配置稀客订单和普客订单的处理阶段。
- 自动化状态会显示厨具预约和送餐盘压力。稀客自动化会锁定本次选择的料理/酒水，并可复用已堆积的可用料理。
- 目标料理或酒水进入送餐盘后会尽快单项送达以释放格子；普客自动化会保留游戏暂存容器链路，并在开启送达阶段时尽快从暂存容器转送订单。
- 自动开始料理时会固定尝试完成原生 QTE 奖励结算。

### 运行时修改

- 修改页可用快捷按钮调整当前运行时材料和酒水库存数量。
- 支持把当前存档可编辑材料或酒水批量设为 99。

### 伴随窗口体验

- 独立伴随窗口可移动、缩放、置顶，并会记住上次的位置和大小，下次启动时自动恢复。
- `F8`/`RS Click` 可在游戏和伴随窗口之间切换，`F10` 可开启或关闭鼠标穿透锁定，游戏关闭后窗口会自动退出。
- 独立伴随窗口支持手柄导航，可用摇杆/十字键移动、确认、返回、切页、滚动和收藏推荐项。
- 设置页可分别调整背景透明度和文字透明度，并可配置置顶状态、主题、焦点切换行为、BepInEx 原生日志窗口、缺失厨具过滤、任务料理优先、推荐排序、同基础料理展示数量、实验性游戏界面置顶和目标厨具高亮。

### 日志、帮助与排查

- 概览和经营中页会显示关键快照耗时，用于排查运行时读取或自动化导致的卡顿。
- 日志页读取 `BepInEx/LogOutput.log` 和自动化作业日志尾部内容并限制缓存上限，可按需开启经营诊断，也可以一键导出诊断包。
- 帮助页使用折叠面板展示功能用法、常见问题和排查步骤。
- Mod 默认关闭下一次启动的 BepInEx 控制台日志窗口，并在当前 Windows 会话尝试隐藏控制台；需要临时查看时可在设置页重新开启。

### 当前限制

- 特殊经营挑战暂不做额外规则适配；推荐和自动化保持标准料理、酒水链路，挑战目标与结算以游戏内提示为准。

